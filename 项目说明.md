# 项目说明

## 1. 项目概览

这是一个基于 `Nuxt 4 + Vue 3 + GSAP + Lenis + TypeScript` 的个人数字世界项目，当前目录为：

`D:\personal_profile3`

项目目标不是常规的个人主页，而是一个带有电影感、情绪表达、雾玻璃交互和滚动叙事的沉浸式数字作品。当前实现围绕以下体验展开：

> 坐在森林边的车里，窗外下雨，玻璃起雾；手指擦过后看见景色，过一会儿雾气又重新凝结。

目前页面已经具备：

- 全站统一雾窗背景
- 双图驱动的擦雾交互
- 开场幕布动画
- 中文主体内容
- 多种 section 布局
- GSAP 滚动叙事与卡片显影动画

当前大量文字、人物、照片、食物、旅行、音乐信息仍然是“可替换占位内容”，方便后续替换成你的真实资料。

## 6. 雾窗背景与图片资源说明

### 原始资源

用户原始提供：

- `resource/有雾气.png`
- `resource/没有雾气.png`

### 稳定引用路径

统一从：

- `utils/resourcePaths.ts`

读取：

```ts
export const cinematicWindowResources = {
  foggedWindow: "/resource/fogged-window.png",
  clearWindow: "/resource/clear-window.png"
} as const;
```

### 当前实际使用者

文件：

- `components/interactive/FogGlassCanvas.vue`

它的逻辑是：

- 下层 `img` 显示清晰图
- 上层 `canvas` 绘制有雾图
- 鼠标移动时在 canvas 上用 `destination-out` 连续擦除
- 停止后按轨迹重新凝结

### 非常重要

如果你以后替换雾窗背景图：

1. 优先替换 `app/public/resource/fogged-window.png`
2. 优先替换 `app/public/resource/clear-window.png`
3. 尺寸比例尽量保持一致
4. 不要改 `utils/resourcePaths.ts` 的 URL 命名，避免动画/构建层再出路径问题

## 7. 开场幕布说明

文件：

- `components/interactive/LoadingVeil.vue`

当前实现：

- 初始整屏幕布覆盖主页面
- 幕布内有“毛毛的世界”字样
- 标题先做模糊淡入
- 白线展开
- 幕布上升
- 主页面随后通过 `opacity + blur + y` 做显影过渡

与之配合的全局显示控制在：

- `app/assets/styles/base.css`

使用的是：

```css
html:not([data-intro-ready="true"]) [data-page-root] { ... }
```

也就是说：

- 开场未完成前，主页面整体隐藏
- 幕布结束前后切换 `data-intro-ready`

如果你以后要改开场片头，优先改：

1. `LoadingVeil.vue`
2. `base.css` 里的 `[data-page-root]` 过渡

不要只改其中一处，否则容易出现：

- 幕布上面被主页面文字覆盖
- 幕布结束后内容突然弹出

## 9. 布局系统与 data hooks

`SectionFrame.vue` 是所有内容 section 的底座。

它统一提供：

- `id`
- `eyebrow`
- `kicker`
- `title`
- `description`
- `layout`

当前 layout 类型：

- `immersive`
- `floating`
- `overlap`
- `broken`
- `timeline`
- `sticky`
- `closing`

还会给滚动动画层提供：

- `data-section`
- `data-layout`
- `data-motion-target="section-header"`
- `data-motion-target="section-title"`
- `data-motion-target="section-description"`
- `data-motion-target="section-body"`

各 section 内还有自己的附加 hook，例如：

- `about-card`
- `timeline-item`
- `gallery-card`
- `travel-card`
- `music-card`
- `food-card`

### 很重要

如果你以后改 section 结构：

- 尽量保留现有 `data-motion-target`
- 否则 `ScrollStorytelling.vue` 的 GSAP 动画会找不到目标
