# Codex Agent 分工职责说明（统一工作规范）
这是一个 Codex Agent 分工的说明，该说明旨在将不同的 Codex 对话打造成分工明确、职责清晰、互不干扰的专家团队。每一个 Codex 对话在开始工作之前，都应该首先阅读并严格遵守本说明。

---

# 一、项目核心定位

本项目并不是一个普通的网站开发项目，而是一个具有**电影镜头语言、情绪表达能力与沉浸式交互体验**的个人数字世界。

开发过程中必须始终遵循以下优先级：

> **Emotion（情绪表达） > Experience（用户体验） > Visual（视觉表现） > Function（功能实现）**

如果某项功能会降低整体品质，应优先选择删除，而不是保留。
---

# 二、统一技术规范

项目统一采用以下技术栈：

* Vue3
* Nuxt4
* GSAP
* ScrollTrigger
* Timeline
* SplitText
* Flip
* MotionPath
* DrawSVG
* Lenis

目标浏览器：

* Microsoft Edge（第一优先）
* Google Chrome

所有 Agent 必须保持技术实现一致，不允许私自引入其他动画框架或重复实现已有能力。

---

# 三、整体设计风格

整个项目必须保持统一美术语言：

* 克制
* 忧郁
* 蓝色低饱和
* 雾化玻璃（Fog Glass）
* Apple / Nothing / Linear 风格

严禁出现：

* 赛博朋克风格
* 巨大发光效果
* 夸张霓虹
* 粒子爆炸特效
* 廉价炫光动画

整体目标是营造一种安静、细腻、具有呼吸感的高级交互体验。

---

# 四、资源管理规范

所有占位图片 需要在统一放置于 该目录下创建的新文件夹Photos

所有内容均需预留未来替换接口。

---

# 五、Agent 分工原则

每个 Codex Agent 都是一个独立专家，只负责自己的职责范围。

原则如下：

* 只修改自己负责目录。
* 不跨职责修改其他 Agent 对话的代码。
* 不主动重构其他模块。
* 不因为方便而越权修改。
* 保持接口稳定。
* 保持代码可合并。
* 保持命名统一。
* 每一个agent对话应该遵循自己的工作顺序。
* 每一个agent对话严格遵循自己的定位与工作要求。

# 十、Merge（合并）策略

所有 Agent 必须遵循以下原则：

* Agent1 不修改 Sections。
* Agent2 不修改 Motion。
* Agent3 不修改基础架构。
* Agent4 不主动开发业务，仅负责审查、修复、整合。

任何超出职责范围的修改，都应先说明理由并获得统一确认。

---

# 十一、最终验收标准

项目完成后，应当让用户产生如下感受：

> “我正在观看一部可以随着滚动推进剧情的交互电影。”

而不是：

> “这是一个不错的模板网站。”

所有开发决策，都应围绕这一最终目标展开。
